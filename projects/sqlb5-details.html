Виртуальные таблицы и материализованные представления

Зачем нам нужны виртуальные таблицы и материализованные представления в базах данных? Базы данных - это просто таблицы, не так ли? Нет, не совсем. Базы данных похожи на объектно-ориентированные языки. Помните, в Java у вас много абстракций (мы имеем в виду Java интерфейсы). Нам нужна абстракция для достижения "чистой архитектуры" и изменения объектов с минимальным влиянием на зависимости.

Более того, в реляционных базах данных существует специфический архитектурный шаблон под названием ANSI/SPARK. Этот шаблон делит объекты на три уровня:

  • Внешний уровень
  • Концептуальный уровень
  • Внутренний уровень

Поэтому мы можем сказать, что виртуальные таблицы и материализованные представления являются физическими интерфейсами между таблицами с данными и пользователем/приложением.

Практические упражнения

00

Упражнение 00: Создадим отдельные представления для людей

Создайте 2 представления базы данных (с атрибутами, аналогичными исходной таблице) на основе простой фильтрации по полу людей. Установите соответствующие имена для представлений: v_persons_female и v_persons_male.

-- Решение:
CREATE VIEW v_persons_female AS
SELECT * FROM person WHERE gender = 'female';

CREATE VIEW v_persons_male AS
SELECT * FROM person WHERE gender = 'male';
01

Упражнение 01: От частей к общему представлению

Используйте 2 представления из упражнения #00 и напишите SQL для получения имен женщин и мужчин в одном списке. Упорядочите по имени человека.

-- Решение:
SELECT name FROM v_persons_female
UNION ALL
SELECT name FROM v_persons_male
ORDER BY name;
02

Упражнение 02: "Сохраним" сгенерированные даты в одном месте

Создайте представление базы данных v_generated_dates, которое должно "хранить" сгенерированные даты с 1 по 31 января 2022 года в типе DATE. Не забудьте упорядочить по столбцу generated_date.

-- Решение:
CREATE VIEW v_generated_dates AS
SELECT generate_series('2022-01-01'::date,
    '2022-01-31'::date, '1 day'::interval)::date AS generated_date
ORDER BY generated_date;
03

Упражнение 03: Найдем пропущенные дни посещений с помощью представления

Напишите SQL-запрос, который возвращает пропущенные дни для посещений людей в январе 2022 года. Используйте представление v_generated_dates для этой задачи и отсортируйте результат по столбцу missing_date.

-- Решение:
SELECT generated_date AS missing_date
FROM v_generated_dates
EXCEPT
SELECT visit_date
FROM person_visits
ORDER BY missing_date;
04

Упражнение 04: Найдем что-то из теории множеств

Напишите SQL-запрос, который удовлетворяет формуле (R - S)∪(S - R). Где R - таблица person_visits с фильтром на 2 января 2022 года, S - также таблица person_visits, но с другим фильтром на 6 января 2022 года. Выполните вычисления с множествами по столбцу person_id, и этот столбец будет единственным в результате. Отсортируйте результат по столбцу person_id и представьте ваш окончательный SQL в представлении базы данных v_symmetric_union.

-- Решение:
CREATE VIEW v_symmetric_union AS
(SELECT person_id FROM person_visits
  WHERE visit_date = '2022-01-02'
  EXCEPT
  SELECT person_id FROM person_visits
  WHERE visit_date = '2022-01-06')
UNION
(SELECT person_id FROM person_visits
  WHERE visit_date = '2022-01-06'
  EXCEPT
  SELECT person_id FROM person_visits
  WHERE visit_date = '2022-01-02')
ORDER BY person_id;
05

Упражнение 05: Рассчитаем цену со скидкой для каждого человека

Создайте представление базы данных v_price_with_discount, которое возвращает заказы человека с именем человека, названием пиццы, реальной ценой и вычисляемым столбцом discount_price (с примененной 10% скидкой и удовлетворяющим формуле price - price*0.1). Отсортируйте результат по именам людей и названиям пицц и преобразуйте столбец discount_price в целочисленный тип.

-- Решение:
CREATE VIEW v_price_with_discount AS
SELECT p.name, m.pizza_name, m.price,
    (m.price - m.price * 0.1)::integer AS discount_price
FROM person_order po
JOIN person p ON po.person_id = p.id
JOIN menu m ON po.menu_id = m.id
ORDER BY p.name, m.pizza_name;
06

Упражнение 06: Материализация из виртуализации

Создайте материализованное представление mv_dmitriy_visits_and_eats (с включенными данными) на основе SQL-запроса, который находит название пиццерии, где Дмитрий посетил 8 января 2022 года и мог есть пиццу дешевле 800 рублей.

-- Решение:
CREATE MATERIALIZED VIEW mv_dmitriy_visits_and_eats AS
SELECT pz.name
FROM person_visits pv
JOIN pizzeria pz ON pv.pizzeria_id = pz.id
JOIN menu m ON pz.id = m.pizzeria_id
JOIN person p ON pv.person_id = p.id
WHERE p.name = 'Dmitriy'
  AND pv.visit_date = '2022-01-08'
  AND m.price < 800;
07

Упражнение 07: Обновим наше состояние

Обновите данные в нашем материализованном представлении mv_dmitriy_visits_and_eats из упражнения #06. Перед этим действием создайте еще одно посещение Дмитрия, которое удовлетворяет SQL-условию материализованного представления, кроме пиццерии, которую мы видим в результате из упражнения #06. После добавления нового посещения обновите состояние данных для mv_dmitriy_visits_and_eats.

-- Решение:
-- Сначала добавим новое посещение
INSERT INTO person_visits (id, person_id, pizzeria_id, visit_date)
VALUES (
  (SELECT MAX(id) + 1 FROM person_visits),
  (SELECT id FROM person WHERE name = 'Dmitriy'),
  (SELECT id FROM pizzeria WHERE name != 'Papa Johns'),
  '2022-01-08'
);

-- Затем обновим материализованное представление
REFRESH MATERIALIZED VIEW mv_dmitriy_visits_and_eats;
08

Упражнение 08: Просто очистим нашу базу данных

После всех наших упражнений у нас есть несколько виртуальных таблиц и материализованное представление. Давайте удалим их!

-- Решение:
DROP VIEW IF EXISTS v_persons_female;
DROP VIEW IF EXISTS v_persons_male;
DROP VIEW IF EXISTS v_generated_dates;
DROP VIEW IF EXISTS v_symmetric_union;
DROP VIEW IF EXISTS v_price_with_discount;
DROP MATERIALIZED VIEW IF EXISTS mv_dmitriy_visits_and_eats;

Концепции представлений

-- Создание представления
CREATE VIEW view_name AS
SELECT ... FROM ... WHERE ...;

-- Создание материализованного представления
CREATE MATERIALIZED VIEW mv_name AS
SELECT ... FROM ... WHERE ...;

-- Обновление материализованного представления
REFRESH MATERIALIZED VIEW mv_name;

-- Удаление представления
DROP VIEW view_name;

-- Удаление материализованного представления
DROP MATERIALIZED VIEW mv_name;
